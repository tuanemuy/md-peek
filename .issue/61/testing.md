# 動作確認計画 — Issue #61: Ctrl+Cで終了できないことがある

**Issue:** #61
**作成日:** 2026-03-22

---

## 確認環境

- ローカル開発環境
- ブラウザ（SSE接続を確立するため）

## 確認項目

### 1. SSE接続がある状態でCtrl+Cで終了できること

- **目的:** 根本原因であるSSE接続によるシャットダウンハングが解消されていることを確認する
- **手順:**
  1. `pnpm dev:server` または `node dist/index.js <markdownファイル>` でサーバーを起動する
  2. ブラウザでプレビューページを開く（SSE接続が確立される）
  3. ターミナルで Ctrl+C を押す
  4. "Shutting down..." → "Server stopped. Bye!" が表示されることを確認する
- **期待結果:** プロセスが即座に（1-2秒以内に）終了する
- **確認ポイント:** プロセスがハングせず、ターミナルにプロンプトが戻ること

### 2. 複数のSSE接続がある状態でCtrl+Cで終了できること

- **目的:** 複数のブラウザタブで接続している場合でも正しく終了することを確認する
- **手順:**
  1. サーバーを起動する
  2. 複数のブラウザタブでプレビューページを開く（3タブ以上推奨）
  3. ターミナルで Ctrl+C を押す
- **期待結果:** プロセスが即座に終了する
- **確認ポイント:** 複数接続があっても遅延なく終了すること

### 3. SSE接続がない状態でCtrl+Cで終了できること

- **目的:** SSE接続がない場合の既存のシャットダウンフローが壊れていないことを確認する
- **手順:**
  1. サーバーを起動する
  2. ブラウザを開かずに、ターミナルで Ctrl+C を押す
- **期待結果:** プロセスが即座に終了する
- **確認ポイント:** 変更前と同じ挙動であること

## エッジケース・異常系

### 1. Ctrl+Cを連打した場合

- **目的:** `shuttingDown` フラグによる二重シャットダウン防止が正しく機能することを確認する
- **手順:**
  1. サーバーを起動し、ブラウザでプレビューを開く
  2. Ctrl+C を素早く2-3回連打する
- **期待結果:** "Shutting down..." メッセージが1回だけ表示され、正常に終了する

## 既存機能への影響確認

- SSE接続によるライブリロードが、シャットダウン前は正常に動作すること（ファイルを変更してブラウザに反映されることを確認）
