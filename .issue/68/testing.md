# 動作確認計画 — Issue #68: 2回目のSIGINTでプロセスを強制終了する

**Issue:** #68
**作成日:** 2026-03-24

---

## 確認環境

- ローカル開発環境
- ターミナルから `pnpm dev` または `pnpm start` でサーバーを起動

## 確認項目

### 1. 通常のシャットダウン（1回目の Ctrl+C）

- **目的:** 既存の正常シャットダウンが壊れていないことを確認
- **手順:**
  1. `pnpm dev -- ./README.md` でサーバーを起動
  2. Ctrl+C を1回押す
- **期待結果:** "Shutting down..." → "Server stopped. Bye!" と表示され、プロセスが終了コード 0 で終了
- **確認ポイント:** エラーメッセージが出ないこと

### 2. 2回目の Ctrl+C で強制終了

- **目的:** 2回目のシグナルで即座にプロセスが終了することを確認
- **手順:**
  1. `pnpm dev -- ./README.md` でサーバーを起動
  2. 別ターミナルで `PID=$(pgrep -f "tsx.*src/index.ts" | head -1)` でPIDを取得（複数マッチする場合があるため `head -1` で絞り込む）
  3. `kill -SIGINT $PID; sleep 0.1; kill -SIGINT $PID` で2回送信（`sleep 0.1` で1回目のシグナル処理を待つ）
  4. 元のターミナルで出力を確認
- **期待結果:** "Force exiting..." と表示され、プロセスが終了コード 1 で終了する
- **確認ポイント:** "Server stopped. Bye!" が表示されない（クリーンアップ完了前に終了するため）

## エッジケース・異常系

### 1. SIGTERM での動作

- **目的:** SIGTERM でも同様に2回目で強制終了することを確認
- **手順:**
  1. サーバーを起動
  2. 別ターミナルから `kill <PID>` を2回素早く実行
- **期待結果:** 2回目の SIGTERM でプロセスが即座に終了

## 既存機能への影響確認

- 1回目の Ctrl+C での正常シャットダウンフローに変更なし
- 既存のユニットテスト（`pnpm test`）がすべてパスすること
