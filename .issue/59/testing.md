# 動作確認計画 — Issue #59: ファイルウォッチャーのエラー時にログを出力する

**Issue:** #59
**作成日:** 2026-03-23

---

## 確認環境

- ローカル開発環境
- Node.js がインストールされていること

## 確認項目

### 1. ファイル監視のエラー時にログが出力される

- **目的:** watcherでエラーが発生した際に `console.warn` でメッセージが出力されることを確認する
- **手順:**
  1. `pnpm dev` でプレビューサーバーを起動し、Markdownファイルを監視させる
  2. 監視対象のファイル/ディレクトリのパーミッションを変更する（`chmod 000`）か、監視対象を削除する
  3. ターミナルの出力を確認する
- **期待結果:** stderrに `File watcher error for ...` または `Directory watcher error for ...` というメッセージが出力される
- **確認ポイント:** メッセージにファイルパスとエラー内容が含まれていること

### 2. 通常のファイル監視が引き続き動作する

- **目的:** ログ出力の追加が通常のファイル監視に影響しないことを確認する
- **手順:**
  1. `pnpm dev` でプレビューサーバーを起動する
  2. 監視対象のMarkdownファイルを編集・保存する
  3. ブラウザのプレビューが更新されることを確認する
- **期待結果:** 変更がリアルタイムに反映される
- **確認ポイント:** エラーなしの場合にwarnが出力されないこと

## エッジケース・異常系

### 1. 監視対象ディレクトリが削除された場合

- **目的:** ディレクトリ削除時のエラーハンドリングを確認する
- **手順:**
  1. `pnpm dev` でディレクトリ監視を開始する
  2. 別ターミナルで監視対象ディレクトリを `rm -rf` で削除する
- **期待結果:** エラーログが出力され、アプリケーションがクラッシュしない

## 既存機能への影響確認

- ファイル変更検知のデバウンス動作が変わっていないこと
- `close()` 後のwatcher動作が変わっていないこと
